IR (Pauli string) grouping

For the IR grouping method, Pauli strings are grouped based on the set of qubit indices where the corresponding Pauli operators are non-identity ('I'). Each group consists of Pauli strings that act non-trivially (i.e., with X, Y, or Z) on the same set of qubits

E.g., list of random Pauli strings

In[2]:
def random_pauli(n: int) -> str:
    return ''.join(np.random.choice(['X', 'Y', 'Z', 'I'], n, replace=True))

def config_to_circuit(config):
    """Convert a BSF-simplified configuration to a Circuit instance"""
    circ = Circuit()
    for item in config:
        if isinstance(item, Clifford2Q):
            circ.append(item.as_gate())
        if isinstance(item, BSF):
            circ += item.as_cnot_circuit(high_level=True)
    return circ

Remark:

  1. For each simplified IR, i.e., subcircuit composed by 1Q Pauli rotation, 2Q Pauli rotation and Clifford2Q gates, left_end_empty_layers is not necessarily equal to right_end_empty_layers, since we are assembling the entire circuit from head to tail, that is, the subcircuit layering is also in a from head-to-tail approach. (e.g., left_end_empty_layers(blk_b) is [3 0 0 0 0 1 2] while right_end_empty_layers(blk_b) is [3 0 2 0 2 1 2])

def plot_tetris_edge(left_end, right_end):
    import matplotlib.pyplot as plt

    offset = 0.1
    xlim = max(left_end) + max(right_end) + 2
    ylim = len(left_end)

    # left_edge_data = [(0, 0)]
    left_edge_data = []
    for i, end in enumerate(left_end):
        left_edge_data.append((end, i))
        left_edge_data.append((end, i + 1))
    # left_edge_data.append((0, ylim))
    left_x, left_y = [], []
    for data in left_edge_data:
        left_x.append(data[0])
        left_y.append(data[1])

    # right_edge_data = [(xlim, 0)]
    right_edge_data = []
    for i, end in enumerate(right_end):
        right_edge_data.append((xlim - end, i))
        right_edge_data.append((xlim - end, i + 1))
    # right_edge_data.append((xlim, ylim))
    right_x, right_y = [], []
    for data in right_edge_data:
        right_x.append(data[0])
        right_y.append(data[1])

    plt.figure(figsize=(3 * (xlim / ylim), 3))
    plt.plot(left_x, left_y, color='coral')
    plt.plot(right_x, right_y, color='coral')
    plt.xlim(-offset, xlim + offset)
    plt.ylim(0, ylim)
    # annotation left_edge_data
    for i, end in enumerate(left_end):
        plt.text(end + 0.3, i + 0.5, f'{end}', ha='center', va='center', fontsize=10, color='C0', fontdict={'weight': 'bold'})
    # annotation at right_edge_data
    for i, end in enumerate(right_end):
        plt.text(xlim - end - 0.3, i + 0.5, f'{end}', ha='center', va='center', fontsize=10, color='C0', fontdict={'weight': 'bold'})
    plt.xticks(range(xlim + 1))
    plt.yticks(range(ylim + 1))
    plt.gca().invert_yaxis()
    plt.grid(True, linestyle='--', alpha=0.7)

    plt.tight_layout()
    plt.show()

In[114]:
def depth_overhead(lhs: np.ndarray, rhs: np.ndarray):
    """Calculate the basic circuit depth(2q) overhead once lhs and rhs are concatenated"""
    xor = np.logical_xor(lhs, rhs)
    if np.all(xor[(lhs == 0) | (rhs == 0)]):
        cost = (lhs + rhs - 1).sum()
    else:
        cost = (lhs + rhs).sum()
    return cost

Comparison with Rustiq

  • In fact, Rustiq is not an end-to-end compiler like Phoenix. We can only calculate the synthesis effect (entangling count and depth statistics) in the logical circuit level. We cannot get perform any hardware mapping given the results acquired by Rustiq, unless complicated modification on Rustiq.
  • Like what the original Rustiq's paper, we set preserver_order=True lead to better results overall.
  • Like what the original Rustiq's paper, we perform evaluation for Rustiq-rcount and Rustiq-rdepth, respectively.

As the following two figure shows, Phoenix significantly outperforms Rustiq overall.

Herein optimization rate is defined as $\frac{\#2Q_{\mathrm{after}}}{\#2Q_{\mathrm{before}}}$. The lower, the better.

  • Num2Q optimization results:
    • optimization rate (Rustiq-rcount) 0.3294
    • optimization rate (Rustiq-rdepth) 0.3956
    • optimization rate (Phoenix) 0.2115
  • Depth2Q optimization results:
    • optimization rate (Rustiq-rcount) 0.2423
    • optimization rate (Rustiq-rdepth) 0.1799
    • optimization rate (Phoenix) 0.1928

from rustiq import pauli_network_synthesis, Metric
from rustiq.utils import entangling_count, entangling_depth, convert_circuit

def rustiq_pass_rcount(paulis):
    circ = pauli_network_synthesis(paulis, Metric.COUNT) # preserve_order is True by default
    # because the returned circuit is a Clifford circuit on one side, we need to double the entangling count and depth
    return entangling_count(circ) * 2, entangling_depth(circ) * 2

def rustiq_pass_rdepth(paulis):
    circ = pauli_network_synthesis(paulis, Metric.DEPTH)
    return entangling_count(circ) * 2, entangling_depth(circ) * 2
